This structure turns frontend development into a highly predictable assembly line. When you need a new feature, you execute these steps in order:

1. Define the Contract (src/types/index.ts)
You write the TypeScript interface that perfectly mirrors the JSON payload your backend is going to send. This is the source of truth.

2. Wire the Network (src/hooks/queries/useJobs.ts)
You write the React Query hook. You tell it to hit your API endpoint and return data matching the interface you just wrote.

3. Generate the UI (src/components/features/...)
This is where you deploy the LLM. You open a new file (e.g., ApplicationTable.tsx), highlight the TypeScript interface from step 1, and prompt the LLM:
"Read this interface. Generate a stateless dumb component using shadcn/ui and Tailwind to display this data in a table. Do not include any data fetching."

4. Connect Them (src/app/...)
You open your page/container component. You import the hook, you import the dumb component, and you plug them together.
